Basics

Variables

A variable is very useful if you do not want to call something over and over again, such as if you want to print one thing over and over.

To create a variable, start by putting "local" (means the same as a variable), the variable name (whatever you want to call the variable as), the equals sign, and then the value you want to do (whatever thing you want such as a string, number, table, etc). 

local variablename = "kkk" -- this value is "kkk" as a string

Here is what code looks like before/after a variable is used. 

print("test")
print("test")
print("test")
print("test")
print("test")

local t = "test"
print(t)
print(t)
print(t)
print(t)
print(t)

You can also change variables as much as you like. 

local t = "test"
print(t)
print(t)
t = "test2"
print(t)
print(t)
print(t)

This would output
test
test
test2
test2
test2

Functions

To create a function, enter in "function", the function name (same thing as a variable name), parenthesis (inside the parenthesis, put the input), write code after, and close it off with end.

Functions do an Input, then an Output. Such as 

function a(text) -- inside the parenthesis put the "input" you want
    print(text)
end

However, the input is optional (if you don't need it) 

function a()
    print("idk")
end

To call a function, you need to put the function name, and then parenthesis after (inside, you put the input.) 

function a(text)
    print(text)
end
a("test")

Which outputs
test

You can write whatever you want in the function, as long as it is closed with an end statement.

You can also make a function return something (outputting something back to you.) 

function a(text)
    print(text)
	return 1
end
local number = a("test")
print(number) -- This will print 1 because that is what the function returned.

The return function will also stop the whole function.

If Statements

If you do not know the operations "<" ">" "==" "~=", then click here

If statements check if something is true, and executes the code from the if statement to the end.

local x = 999
local y = 999
if x == y then
	print("This code should execute!")
end
x = 0
y = 999
if x == y then
	print("This code shouldn't execute!")
end
print("This code will execute no matter what since it is outside the if statement.")

Operations

Operations are useful for if statements, or making something equal to another thing.

You cannot use the Equal Sign for if statements.

Equal Sign Operator (=)

The equal operator makes something equal another thing. 

local s = 123901239012390

Double Equal Sign Operator (==)

The double equal operator returns true when something is equal to another thing. 

local x = 1
local y = 2
if x == y then
	print("This shouldn't execute because x is not equal to y.")
end

Inequality Operator (~=)

The Inequality operator returns true when something is not equal to another thing. 

local x = 1
local y = 2
if x ~= y then
	print("This should execute because x is not equal to y.")
end

Greater Than Operator (>)

The Greater Than operator returns true when something is bigger than another thing. 

local x = 1
local y = 2
if x > y then
	print("This shouldn't execute because x is not bigger than y.")
end

Smaller Than Operator (<)

The Smaller Than operator returns true when something is smaller than another thing. 

local x = 1
local y = 2
if x < y then
	print("This should execute because x is smaller than y.")
end

Bigger Than or Equal To Operator (>=)

The Bigger Than or Equal To operator returns true when something is bigger than another thing or equals to the other thing. 

local x = 1
local y = 2
if x >= y then
	print("This shouldn't execute because x is not bigger or equal to y.")
end

Smaller Than or Equal To Operator (<=)

The Smaller Than or Equal To operator returns true when something is smaller than another thing or equals to the other thing. 

local x = 1
local y = 2
if x <= y then
	print("This should execute because x is smaller than y (however not equal to.)")
end

For Statements

A for statement has a Numeric version (looping a number) and a Generic version (looping through a table.)

Numeric for statement

A numeric for statement works like this:

Customizable variables (updated with the loop number each iteration/loop): 

for i

Minimum & Maximum: 

for i = 5,10

Increments (default is 1): 

for i = 5,10,1

A Do statement (mostly used in for statements so not really much about it): 

for i = 5,10,1 do

Code written inside: 

for i = 5,10,1 do
	print(i)

An end statement to close it off:

And the final result is this 

for i = 5,10,1 do -- you can leave the last number (1) blank since the for loop already defaults to 1. So this line would look like for i = 5,10 do
	print(i)
end

Which outputs
5
6
7
8
9
10

If we change the incement of the for statement like this 

for i = 5,10,2 do
	print(i)
end

It will output
5
7
9

This is because every iteration/loop, it checks if the number is bigger or smaller than the maximum/minimum. If its bigger/smaller, then the loop will end, and continue on with the other code not inside the loop.

Generic for statement

A generic for statement loops through a table, updating the customizable variables each loop. It is structured like this:

The index and value of the table: 

for i,v

The in statement (this is needed, but it does nothing): 

for i,v in

The pairs statement (gets both the looped index and values in a table, using ipairs is the same but I just use pairs): 

for i,v in pairs

Parenthesis after the pairs statement to complete it (put the table you want to loop through inside the parenthesis):

if you are wondering what the pairs statement does, it basically just returns both the index and value. 

for i,v in pairs({"a","b","c"})

A Do statement (mostly used in for statements so not really much about it): 

for i,v in pairs({"a","b","c"}) do

Code written inside: 

for i,v in pairs({"a","b","c"}) do
	print(i,v)

An end statement to close it off:

And the final result is this 

for i,v in pairs({"a","b","c"}) do
	print(i,v)
end

This should output
1 a
2 b
3 c

Tables

Tables are useful for storing alot of data in one without having to make alot of variables.

To simply make a table, put a opening backet {, then put data you want sepeating them with commas {1,2,3,4,"yes","no" and then close it off with an ending bracket }.
The result is this 

local thing = {1,2,3,4,"yes","no"}

To get data from the table, simply get it by the Index value.

if you are setting the table like this: 

local thing = {1,2,3,4,"yes","no"}

Then lua will automatically number the index starting at 1, and continue to increase by 1 each comma. So if you want to get the "yes" value, it would be at the 5th index. 

local thing = {1,2,3,4,"yes","no"}
print(thing[5]) -- this should print "yes"

You can also set the index value manually like this: 

local thing = {}
thing["s"] = 99

Or like this 

local thing = {}
thing.s = 99 -- quick note, you cannot put a number after the comma otherwise it will error (but you can put a number after the letter after the comma like this: thing.s2 = 99). This only applies to when you are using the dot. You can index it like this: thing[2] = 99

Either way, they still have the same result. 

local thing = {}
thing.s = 99
print(thing.s) -- should print 99
local thing = {}
thing["s"] = 99
print(thing["s"]) -- should print 99

Data Types

Data types are useful if you need to declare a certain variable.

This is how to make a string. 

local test = "jfkldsjflksdjklfdsjklfsd"

Strings can also be concatenated/merged with another string with 2 dots (..) 

local test = "you are ".."gay"
print(test) -- should print "you are gay"

This is how to make a number. 

local test = 88888888

You can also do math functions on a number. 

local test = 88888888
test = test - test -- should equal 0 since it is subtracted by itself. (note: you can also combine the - and = sign by doing -=, for example: test -= test will do the same thing, just shorten it. But doing that on normal lua (not roblox's) will error. The trick also works for other math things like += /= *= etc.)

This is how to make a UDim2 (for GUI's), Structured as - X Scale, X Offset, Y Scale, Y Offset. 

local test = UDim2.new(1,0,1,0)

This is how to make a Vector3 (Object Size/Object Position), Structured as - X, Y, Z. 

local test = Vector3.new(10,10,10)

This is how to make a CFrame (Position & Object Rotation), Structured as - X, Y, Z. There are also more other ways to structure but it's complicated. Read it at this website. 

local test = CFrame.new(1728178,8,1)

This is how to make a table (Data inside of brackets, commas to split the data) 

local test = {1, 2, 3, 4, "aww", "you", "mad"} -- you can change what you want inside the brackets as long as the individual data is split with commas

If you want to read more about tables, click here.

This is how to make a Instance (Roblox parts/models/meshes etc.) Put the ClassName (what type of object) inside the parenthesis as a string. 

local part = Instance.new("Part")

Extras

Tostring

The tostring function converts almost everything to a string, but its use is mostly for numbers. 

local test = 100
test = tostring(test)
local players = "There are "..test.." Players!" -- should print "There are 100 Players!" Without the tostring, the code would error because you cannot merge a string with a number.

Tonumber

The tostring function converts almost everything to a number, but its use is mostly for strings. 

local test = "100"
test = tonumber(test)
test += 10 -- test is now 110
test = tostring(test)
local players = "There are "..test.." Players!" -- should print "There are 110 Players!" Without the tonumber, you couldn't convert the string to a number you can use math functions on and back.
